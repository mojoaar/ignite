How It Works

1. Interview

Ignite conducts a natural 5-phase AI conversation: Identity, Tech Stack, Features, Architecture, and Generation. Answer at your pace — all progress is saved.

2. Generate

Choose your AI provider and model. Watch as Ignite generates four professional documentation files: project spec, agent guide, implementation plan, and README.

3. Ship

Output files written to your project directory. Switch providers mid-conversation. Resume any project from history. Export chats as markdown. Ready to ship.

Output That Ships With Your Project

Every provisioned project gets four files that match professional open-source documentation standards. Your contributors and AI assistants will thank you.

project.md

Full specification: identity, core values, tech stack table, API endpoints, database schema, performance targets, risks & mitigations. Everything a new contributor needs to understand the project in one file.

agents.md

Agent guide for AI coding assistants: quick commands, code rules (always/never), banned packages list, package audit process, architecture decisions, testing commands.

plan.md

Implementation roadmap: phases 0-4 with checkboxes, verification commands, performance targets, risks table, versioning strategy, changelog. Ready for your project manager.

README.md

Polished public README: tagline, badges, features list, quick start commands, development workflow, tech stack, changelog, license. GitHub-ready.
